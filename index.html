06 / DEEP DIVE

Making a streaming pipeline testable when the data feed costs money

Case study · realtime-options-flow

Problem
The pipeline's whole point is a paid, authenticated, real-time market-data WebSocket — which doesn't exist on a CI runner. Mocking the transport encodes today's guess about the wire format and rots silently the moment the feed drifts.
Approach
Split the producer into two interchangeable sources behind one event contract: a live Polygon feed and a synthetic generator that emits the same raw OCC-symbol wire format. Both run the identical parse → enrich → publish path; malformed events route to a dead-letter queue.
Result
One flag switches sources. CI runs the real Kafka + Spark job on the synthetic feed — 200 events/sec, no key, no market hours, no flake — exercising the actual parsing and math on every push. Anyone can make up and watch it work in minutes.

Quants — AI-native risk engine

Ask risk questions in plain English and get institutional-grade analytics back. It now includes a full buy-side risk workbench — benchmark-relative risk (tracking error, active exposures), risk & return attribution, and a constraint / cost / Black-Litterman optimizer — and its analytics core is cross-checked against QuantLib (the engine ORE is built on) to machine precision.
